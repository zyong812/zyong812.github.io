1. Introduction

Clustering problems a very common and classical problem in many fields. K-means is the simplest cluster algorithm, and it is commonly-used but has many limitations. In this report, I did a survey on spectral clustering algorithms. And by experimental testing, it has much better performance than k-means

The main reference for this report is [2]. And I have also made some analysis and improvements for the algorithm by my own, which includes:

  • A explanation for why clustering on spectral can cluster the original data points (In Section 2).
  • A strategy to adjust σ\sigmaσ (Gaussian similarity function parameter) automatically (In section 4.2) for different data points scale.
  • A strategy to select number of clusters kkk according to eigengaps (In section 4.3).

The improved clustering algorithm can run fully automatically without tuning any parameters. I've tested it over several dataset, and it has fine acceptable results. This algorithm is the best clustering algorithm I've found till now.

2. Problem statement

Given a set of data points x1,x2...xnx_1, x_2 ... x_nx1​,x2​...xn​, we want to partition them into kkk clusters, such that for data points in the same cluster they are similar, and for data points in different clusters they are not similarity to each other.

From the perspective of graph theory, the clustering problems can be reformulated into a graph cut problem. If we regard each data points as a vertex, and the similarity measurement as the edge weight between vertices, then we have an undirected graph G=(V,E)G = (V, E)G=(V,E), and the clustering problem is to find a partition for graph GGG.

For computation, a graph is usually represented with a affinity matrix WWW, for which the row or column indices represent vertices, and wij=wjiw_{ij}=w_{ji}wij​=wji​ represents the similarity between vertex iii and vertex jjj. There are many ways to measure the similarity between vertices [2]. The first idea is to make wij=1w_{ij}=1wij​=1 if the distance between vertex iii and jjj is less than a threshhold, otherwise wij=0w_{ij}=0wij​=0. The second idea is based on kkk-nearest neighborhood, for example wij=1w_{ij}=1wij​=1 if vertex jjj is among kkk nearest vertices of iii, otherwise wij=0w_{ij}=0wij​=0. Another idea is to compute the similarity using Gaussian function wij=exp(−∣∣xi−xj∣∣22σ2)w_{ij}=exp(-\frac{||x_i - x_j||^2}{2 \sigma^2} )wij​=exp(−2σ2∣∣xi​−xj​∣∣2​), which will construct a fully connected graph. (I've used the combination of the first and the second idea for implementation, which will described in Section 4.2)

We can define similarity sum SSS as: for two groups of vertex set A1,A2A_1, A_2A1​,A2​ (can be the same set),

S=∑i∈A1,j∈A2wijS = \sum_{i \in A_1, j \in A_2} w_{ij} S=i∈A1​,j∈A2​∑​wij​

For a well-partitioned GGG, the similarity sum for the same group of vertices should be small, and the similarity sum for different groups should be large.

The above problem is a multi-objective and discrete optimization problem, which has been proved to NP-hard. *K-means} is a commonly-used and simple algorithm to solve this problem, but it has obvious limitations. It can be easily stuck in some local minimum, and there is no implication for how to choose kkk automatically. Moreover, for complex data points (e.g. circle-in-circle data points), usually it cannot find good solution.

Here is my understanding. Recall that the eigenspace of eigenvalue 000 is spanned by 1A1,...,1Ak1_{A_1}, ..., 1_{A_k}1A1​​,...,1Ak​​, therefore each column of UUU is a linear combination of 1A1,...,1Ak1_{A_1}, ..., 1_{A_k}1A1​​,...,1Ak​​. where C∈ℜkC \in \Re^kC∈ℜk is the linear combination coefficients matrix. Considering the rows of UUU, they are linear combination of rows of CCC. Since each row of AAA has only one element equal to 1 and other elements are zeros, each row of UUU is actually one of the rows of CCC. Therefore, rows of CCC can be obtained by clustering rows of UUU. Choosing numbers of clusters kkk automatically

As the mathematical analysis implies, the multiplicity of eigenvalue 0 is the number of clusters. However, in practice, usually the similarity is fully connected if we use Gaussian function to construct the similarity matrix. Therefore, we should consider several smallest eigenvalues to decide the cluster numbers kkk. One of the way is two decide the number of kkk by analyzing the eigengaps ∣λi+1−λi∣|\lambda_{i+1} - \lambda_{i}|∣λi+1​−λi​∣.

I've used a strategy to automatically choose kkk according to the eigengap. Suppose we have NNN data points, we can compute the average eigengap as λmax−λminN\frac{\lambda_{max} - \lambda_{min} }{N}Nλmax​−λmin​​. Then, from the smallest eigenvalue λ1=0\lambda_1=0λ1​=0, we campare the eigengap ∣λi+1−λi∣|\lambda_{i+1} - \lambda_{i}|∣λi+1​−λi​∣ with the average eigengap. Until the first time ∣λi+1−λi∣|\lambda_{i+1} - \lambda_{i}|∣λi+1​−λi​∣ is greater than the average eigengap, we stop the process and set k=ik=ik=i.

Figure 8 is the result when using the above strategy to choose kkk automatically. I've test such strategy over other dataset, and it can also have acceptable results. The above strategy works well for some dataset, although it is not perfect. It provides a option for us if we want the clustering process runs fully automatically.

5. Potential applications

This is the best clustering algorithm I've used till now. I am doing some research on video classification. One basic idea is to do classification for each frame of the video, based on which we can categorize the video, but it is too computationally expensive. Considering the fact that most videos are consist of several scenes, which means many consecutive image frames of the video are taken under the similar situation (e.g. scenary video), we can only process one image for each scene. The problem is how to cluster all the image frames into scenes, and the key is to use a good clustering algorithm.

The algorithm introduced in this report is very promising to solve the video scene clustering problem. There are many mature methods to measure the similarity between images (e.g. using ImageNet model to extract features), so it is easy to build the similarity graph. The proposed algorithm can detect a rough kkk, which is enough because usually for video classification we don't need a very precise number of clusters. Therefore, the whole process can run totally automatically.
